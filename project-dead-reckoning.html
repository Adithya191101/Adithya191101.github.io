IMU & GPS-Based Dead Reckoning Navigation

Implementation of a complete navigation stack combining IMU and GPS data through sensor fusion, magnetometer calibration, and dead reckoning to achieve accurate vehicle trajectory estimation.

October 31, 2024
Northeastern University

Sensor Fusion & Localization
View on GitHub View Report (PDF)

Project Overview

This project implements a complete dead reckoning navigation system that fuses IMU (Inertial Measurement Unit) and GPS data to estimate vehicle trajectory. Dead reckoning is a fundamental technique in robotics navigation that computes position by integrating velocity and heading over time, with periodic GPS corrections to bound accumulated errors.

The implementation addresses the inherent challenges of inertial navigation including sensor drift, magnetic distortions, and integration errors through a multi-stage processing pipeline involving magnetometer calibration, complementary filtering for yaw estimation, and GPS-aided trajectory correction.

Learning Objectives

Implement magnetometer calibration techniques, develop sensor fusion algorithms using complementary filters, understand dead reckoning principles, integrate GPS corrections for drift mitigation, and apply these concepts to real-world driving data.

Key Concepts

The project integrates several fundamental concepts in robotic navigation:

Dead Reckoning

Position estimation by integrating velocity and heading from a known starting point

Sensor Fusion

Combining multiple sensor measurements to achieve better accuracy than individual sensors

Calibration

Correcting systematic sensor errors including hard-iron and soft-iron distortions

Drift Correction

Using GPS updates to bound accumulated integration errors over time

Data Collection Scenarios

IMU and GPS data were collected under two different scenarios in the Boston area:

Circular Path (Calibration)

Controlled circular driving pattern ("donut" path) specifically designed for magnetometer calibration. The circular motion ensures all heading directions are sampled, enabling accurate estimation of hard-iron and soft-iron correction parameters.

Urban Navigation (Testing)

Real-world driving data collected near Northeastern University's Ruggles Station area in Boston. This dataset includes various turns, stops, and straight segments representative of typical urban navigation scenarios.

Processing Pipeline

The navigation system is implemented through four sequential MATLAB scripts, each handling a specific stage of the processing pipeline:

1

Magnetometer Calibration

magnetometer_calibration.m

Corrects magnetic sensor distortions using the circular path calibration data. Implements hard-iron offset removal and soft-iron scale factor correction to transform elliptical magnetometer readings into a centered circle.

2

Yaw Estimation (Sensor Fusion)

sensor_fusion.m

Implements a complementary filter that fuses gyroscope angular rates with magnetometer heading. Balances the gyroscope's short-term accuracy with the magnetometer's long-term stability to produce drift-free yaw estimates.

3

Forward Velocity Estimation

fwd_vel_est.m

Derives vehicle forward velocity by integrating accelerometer measurements along the direction of travel. Uses GPS velocity as ground truth for validation and applies corrections to minimize drift accumulation.

4

Dead Reckoning Integration

deadrocking.m

Computes the complete vehicle trajectory by integrating velocity and orientation estimates. Incorporates periodic GPS position fixes to correct accumulated drift and maintain trajectory accuracy over extended distances.

Magnetometer Calibration

Accurate heading estimation requires correcting magnetometer measurements for environmental magnetic distortions:

Hard-Iron Correction

Removes constant magnetic offsets caused by permanently magnetized materials near the sensor (e.g., vehicle chassis, mounting hardware). These offsets shift the center of the magnetometer's circular response away from the origin.

Soft-Iron Correction

Corrects for scale factor differences and axis coupling caused by magnetically permeable materials that distort the local magnetic field. These effects transform the ideal circular response into an ellipse.

Raw Magnetometer Data - Off-center elliptical pattern showing hard-iron offset and soft-iron distortion
Corrected Magnetometer Data - Centered circular pattern after hard-iron and soft-iron calibration

Complementary Filter Design

The yaw estimation combines gyroscope and magnetometer data using a complementary filter structure:

G

Gyroscope (High-Pass)

Provides accurate short-term angular rate measurements. Integration gives heading change but accumulates drift over time. The high-pass characteristic preserves rapid heading changes while attenuating low-frequency drift.

M

Magnetometer (Low-Pass)

Provides absolute heading reference but is susceptible to high-frequency noise and magnetic interference. The low-pass characteristic extracts the stable heading trend while filtering out transient disturbances.

+

Fused Output

The complementary sum produces a yaw estimate that combines the best characteristics of both sensors: responsive to quick turns (gyroscope) while maintaining long-term accuracy (magnetometer).
