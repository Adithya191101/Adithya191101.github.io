GPS Data Analysis & Visualization

A comprehensive GPS data analysis project using ROS2, processing GPGGA NMEA sentences to compute statistical metrics including RMSE, variance, and bias across stationary and mobile scenarios.

September 26, 2024
Northeastern University
EECE 5554 - Robotics Sensing & Navigation
GPS & Sensor Analysis
View on GitHub View Report (PDF)

Project Overview

This laboratory project focuses on GPS data acquisition, processing, and analysis as part of the Robotics Sensing and Navigation course at Northeastern University. Using a USB-based GPS module (BU-353S4), raw GPGGA NMEA sentences were collected and processed to understand GPS accuracy and error characteristics.

The project involves developing a ROS2-based GPS driver to parse NMEA data, convert coordinates to UTM (Universal Transverse Mercator) format, and perform comprehensive statistical analysis including Root Mean Square Error (RMSE), variance, mean, and bias calculations.

Learning Objectives

Understand GPS error characteristics, implement ROS2 sensor drivers, perform coordinate transformations (Lat/Long to UTM), and conduct statistical analysis on real-world sensor data.

Data Collection Scenarios

GPS data was collected under three different scenarios to analyze positioning accuracy and error patterns:

Stationary Data

GPS receiver placed at a fixed known location for extended duration to measure positioning drift and static accuracy characteristics.

Walking Straight Line

Controlled walking along a straight path to evaluate GPS tracking accuracy during linear motion with predictable trajectory.

Dynamic Walking

Free-form walking pattern in open environment to assess GPS performance during varied movement and direction changes.

Implementation Details

The project involved developing a complete ROS2-based pipeline for GPS data acquisition and analysis.

1

ROS2 GPS Driver Development

Created a custom ROS2 node to interface with the BU-353S4 GPS receiver via serial port, parsing GPGGA NMEA sentences and publishing structured GPS messages.

2

NMEA Sentence Parsing

Implemented parser for GPGGA sentences extracting latitude, longitude, altitude, fix quality, number of satellites, and HDOP (Horizontal Dilution of Precision).

3

UTM Coordinate Conversion

Converted WGS84 geographic coordinates (latitude/longitude) to UTM (Universal Transverse Mercator) projection for accurate distance measurements in meters.

4

Data Logging with ROS2 Bags

Recorded GPS data using ROS2 bag files stored in SQLite database format (.db3) for persistent storage and offline analysis.

5

Statistical Analysis & Visualization

Developed Python analysis scripts using NumPy and Matplotlib to compute statistical metrics and generate comprehensive visualizations of GPS performance.
