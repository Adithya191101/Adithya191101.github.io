IMU Noise Characterization with Allan Variance

Comprehensive analysis of IMU sensor noise characteristics using Allan Variance methodology, examining accelerometer, gyroscope, and magnetometer data across short-term and long-term collection periods.

October 14, 2024
Northeastern University
EECE 5554 - Robotics Sensing & Navigation
IMU & Sensor Analysis
View on GitHub View Report (PDF)

Project Overview

This laboratory project focuses on characterizing IMU (Inertial Measurement Unit) sensor noise using Allan Variance analysis. The study examines data from a 9-DOF IMU containing a 3-axis accelerometer, 3-axis gyroscope, and 3-axis magnetometer to understand noise characteristics critical for navigation and state estimation applications.

Allan Variance is a powerful statistical tool originally developed for analyzing frequency stability in oscillators, now widely used in IMU calibration and error modeling. By analyzing sensor data over varying time intervals, it reveals different noise components including white noise, bias instability, and random walk.

Learning Objectives

Understand IMU noise characteristics, implement Allan Variance analysis, develop ROS2 IMU drivers, characterize sensor bias and random walk parameters, and apply statistical methods to real-world inertial sensor data.

Data Collection Scenarios

IMU data was collected under two different duration scenarios to capture both short-term noise and long-term stability characteristics:

15-Minute Dataset

Short-term data collection for initial statistical analysis, histogram generation, and quick noise characterization. Captures high-frequency noise components and sensor behavior during typical operation periods.

5-Hour Dataset

Extended data collection for Allan Variance analysis, enabling identification of bias instability, random walk parameters, and long-term drift characteristics essential for navigation applications.

Short-Term Statistical Analysis (15 min)

Analysis of the 15-minute dataset provides insights into immediate sensor noise characteristics through time-series visualization and histogram distributions.

Implementation Details

The project involved developing a complete ROS2-based pipeline for IMU data acquisition and noise analysis.

1

ROS2 IMU Driver Development

Created a custom ROS2 node to interface with the 9-DOF IMU sensor, publishing standardized sensor_msgs/Imu messages containing linear acceleration, angular velocity, and orientation data.

2

Data Acquisition & Unit Conversion

Implemented proper unit conversions for all sensor axes. Magnetometer readings collected in Gauss were converted to Tesla for standardization. Angular rates converted to rad/s and accelerations to m/s^2.

3

Long-Duration Data Logging

Recorded 5+ hours of stationary IMU data using ROS2 bags for Allan Variance analysis, ensuring stable temperature and minimal vibration during collection.

4

Statistical Analysis (15-min)

Developed Python analysis scripts to compute mean, variance, standard deviation, and generate histograms for each sensor axis using the short-term dataset.

5

Allan Variance Computation

Implemented MATLAB scripts to compute Allan Deviation across multiple averaging times (tau), generating log-log plots to identify noise components from slope analysis.
