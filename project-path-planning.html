Path-Planning Algorithm Benchmarking

Comprehensive benchmarking study comparing sampling-based path planning algorithms including RRT, RRT*, Bidirectional RRT, and Bidirectional RRT* across multiple environment configurations.

April 2024
PSG College of Technology
Mobile Robotics
Motion Planning
View on GitHub View Report (PDF)

Project Overview

Path planning is a fundamental problem in mobile robotics, requiring algorithms that can efficiently find collision-free paths from a start position to a goal in environments with obstacles. This project implements and benchmarks four sampling-based path planning algorithms to evaluate their performance characteristics.

The study focuses on Rapidly-exploring Random Trees (RRT) and their variants, which are particularly effective for high-dimensional configuration spaces and complex environments. Each algorithm was tested across three different map configurations with varying obstacle densities and arrangements.

Project Objectives

  • Implement RRT, RRT*, BRRT, and BRRT* algorithms from scratch in Python
  • Develop visualization tools to animate tree expansion and path finding
  • Compare algorithm performance across different environment configurations
  • Analyze trade-offs between computation time, path optimality, and exploration efficiency

Algorithms Implemented

Four sampling-based path planning algorithms were implemented and compared:

RRT (Rapidly-exploring Random Tree)

The basic RRT algorithm grows a tree from the start position by randomly sampling the configuration space and extending the tree toward each sample. It provides probabilistic completeness but not optimal paths.

Probabilistically Complete Fast Exploration Non-Optimal Paths

RRT* (Optimal RRT)

An optimized version of RRT that rewires the tree to find shorter paths. It includes a rewiring step that checks if nearby nodes can be reached more efficiently through the new node, converging to optimal solutions.

Asymptotically Optimal Path Rewiring Higher Computation

BRRT (Bidirectional RRT)

Grows two trees simultaneously - one from start and one from goal - attempting to connect them. This bidirectional approach significantly reduces exploration time in many environments.

Dual Tree Growth Faster Convergence Connect Heuristic

BRRT* (Bidirectional RRT*)

Combines the bidirectional approach with RRT* optimization. Both trees undergo rewiring, and the connection between trees is optimized, providing both speed and path quality benefits.

Optimal + Bidirectional Best Path Quality Balanced Trade-off

RRT Algorithm Workflow

The core RRT algorithm follows these steps to explore the configuration space:

1

Random Sampling

Generate a random point (x_rand) in the configuration space. With a small probability, sample the goal directly to bias exploration toward the target.

2

Nearest Neighbor Search

Find the node (x_near) in the existing tree that is closest to the random sample using Euclidean distance metric.

3

Tree Extension

Create a new node (x_new) by extending from x_near toward x_rand by a fixed step size. This limits the maximum edge length in the tree.

4

Collision Detection

Check if the edge from x_near to x_new is collision-free by sampling points along the line and testing against obstacle boundaries.

5

Node Addition

If collision-free, add x_new to the tree with x_near as its parent. For RRT*, also perform rewiring to optimize local paths.

6

Goal Check

Check if x_new is within goal radius. If yes, trace back through parent pointers to extract the complete path from start to goal.

Test Environments

Three different 2D environment configurations were created to evaluate algorithm performance under varying conditions:

Map 1: Complex maze-like environment with narrow passages
Map 2: Moderate obstacle density with open spaces
Map 3: Sparse obstacles requiring efficient exploration

Environment Parameters

  • Configuration Space: 2D continuous space with binary occupancy grid
  • Obstacles: Represented as black regions in the map images
  • Start/Goal: Green (start) and red (goal) markers in visualizations
  • Step Size: Fixed extension distance for tree growth
  • Goal Radius: Tolerance for reaching the goal position

RRT Algorithm Results

The basic RRT algorithm explores the space rapidly but produces non-optimal paths. The tree grows outward from the start, eventually finding a path to the goal.

RRT exploration on Map 1 - navigating complex passages
RRT exploration on Map 2 - moderate obstacle avoidance
RRT exploration on Map 3 - efficient open space coverage

RRT Observations

  • Fast initial exploration with wide coverage of the configuration space
  • Paths tend to be jagged and suboptimal due to random sampling nature
  • Performance varies significantly with obstacle density and start/goal positions
  • No path optimization - first valid path found is returned

RRT* Algorithm Results

RRT* adds a rewiring step that continuously improves path quality. As the tree grows, it checks if existing nodes can be reached more efficiently through newly added nodes.

RRT* on Map 1 - optimizing paths through rewiring
RRT* on Map 2 - improved path quality visible
RRT* on Map 3 - near-optimal path generation

RRT* Observations

  • Significantly smoother and shorter paths compared to basic RRT
  • Asymptotically optimal - path quality improves with more iterations
  • Higher computational cost due to rewiring operations
  • Better suited for applications where path quality is critical

BRRT Algorithm Results

Bidirectional RRT grows two trees simultaneously - one from start and one from goal - attempting to connect them in the middle. This approach often finds paths faster than single-tree methods.

BRRT on Map 1 - dual tree growth from start and goal
BRRT on Map 2 - trees connecting in open space
BRRT on Map 3 - efficient bidirectional exploration

BRRT Observations

  • Faster path discovery due to bidirectional exploration
  • More efficient in environments with clear paths between start and goal
  • Tree connection heuristic reduces total exploration needed
  • Path quality similar to basic RRT - not optimal

BRRT* Algorithm Results

Bidirectional RRT* combines the speed advantage of bidirectional search with the path quality optimization of RRT*. Both trees undergo rewiring, producing optimal paths efficiently.

BRRT* on Map 1 - optimized bidirectional planning
BRRT* on Map 2 - smooth path with fast convergence
BRRT* on Map 3 - best of both approaches

BRRT* Observations

  • Best overall performance combining speed and path quality
  • Both trees optimized through rewiring operations
  • Connection point is also optimized for shortest total path
  • Recommended for most practical applications

Algorithm Comparison

Summary of key characteristics across all four algorithms:

Algorithm Path Optimality Convergence Speed Computation Cost Best Use Case
RRT Non-optimal Medium Low Quick feasibility check
RRT* Asymptotically Optimal Slow High High-quality path needed
BRRT Non-optimal Fast Medium Quick path finding
BRRT* Asymptotically Optimal Fast High Optimal path quickly

Key Findings

  • BRRT* provides the best balance of speed and path quality for most scenarios
  • RRT is sufficient when any valid path is acceptable and speed is critical
  • RRT* excels in applications requiring optimal paths with no time constraints
  • BRRT offers fast convergence when path optimality is not the primary concern
  • Bidirectional variants consistently outperform single-tree versions in convergence speed
