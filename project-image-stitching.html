Image Stitching & Panorama Creation

Implementation of a complete image stitching pipeline using Harris corner detection, feature matching with Lowe's ratio test, homography estimation, and seamless blending to create panoramic images.

November 14, 2024
Northeastern University
EECE 5554 - Robotics Sensing & Navigation
Computer Vision
View on GitHub View Report (PDF)

Project Overview

This project implements a complete image stitching pipeline to create seamless panoramic images from multiple overlapping photographs. Image stitching is a fundamental computer vision technique used in applications ranging from smartphone panoramas to satellite imagery and autonomous vehicle perception.

The implementation uses the classical Harris corner detector for feature extraction, followed by descriptor matching, homography estimation using RANSAC, perspective warping, and alpha blending to produce visually seamless panoramas.

Learning Objectives

Implement Harris corner detection from scratch, understand feature descriptor matching with ratio test filtering, estimate homography transformations, apply perspective warping, and blend images seamlessly to create panoramas.

Image Stitching Pipeline

The panorama creation follows a multi-stage computer vision pipeline:

1

Harris Corner Detection

Detect interest points (corners) in each image using the Harris corner detector. Corners are identified where the image gradient changes significantly in multiple directions, making them distinctive and repeatable across views.

2

Feature Description

Extract local descriptors around each detected corner to create a unique signature. These descriptors capture the local image structure and enable matching corners across different images.

3

Feature Matching

Match descriptors between image pairs using Brute-Force matching with L2 norm. Apply Lowe's ratio test (threshold 0.75) to filter out ambiguous matches and retain only reliable correspondences.

4

Homography Estimation

Compute the 3×3 homography matrix that maps points from one image to another using RANSAC to robustly handle outlier matches. This transformation accounts for camera rotation and perspective changes.

5

Perspective Warping

Warp source images into the reference coordinate frame using the computed homography. Automatic padding ensures no content is clipped during the transformation.

6

Image Blending

Blend overlapping regions using weighted averaging to create smooth transitions. Alpha blending eliminates visible seams and produces a seamless final panorama.

Harris Corner Detection

The Harris corner detector is a classical algorithm that identifies points where the image intensity changes significantly in multiple directions:

Algorithm Steps

  1. Compute image gradients Ix and Iy using Sobel operators
  2. Calculate gradient products: Ix², Iy², and IxIy
  3. Apply Gaussian smoothing to gradient products
  4. Compute structure tensor M at each pixel
  5. Calculate corner response R = det(M) - k·trace(M)²
  6. Apply non-maximum suppression to find local maxima
  7. Threshold and select top N strongest corners

Why Corners?

Corners are ideal features for matching because they are distinctive (unique local structure), repeatable (detected consistently across viewpoints), and well-localized (precise position can be determined). Edges and flat regions lack these properties.

Datasets

The stitching pipeline was tested on three different image sequences with varying complexity:

Ruggles Station - Graffiti Mural (7 Images)

Colorful graffiti mural captured near Northeastern University. Rich texture and strong geometric features provide abundant corner detections for reliable matching.

7 overlapping images (50% overlap) of the graffiti mural

Cinder Block (5 Images)

Close-up texture sequence of a concrete/cinder block surface. Tests the algorithm's ability to handle sparse, subtle textures with minimal distinctive features.

5 images of sparse concrete texture - challenging for feature detection

Gallery Wall - 15% Overlap (5 Images)

Framed artwork on a gallery wall with minimal 15% overlap between images. Tests the algorithm's limits with reduced correspondence regions.

5 images with only 15% overlap - testing algorithm robustness

Key Observations

  • Harris corners provide stable, repeatable features across different viewpoints
  • Lowe's ratio test effectively filters ambiguous matches
  • RANSAC robustly estimates homography despite outlier correspondences
  • Alpha blending produces visually seamless transitions in overlap regions
  • Pipeline handles both richly textured scenes (graffiti) and sparse textures (cinder block)
  • Algorithm successfully stitches images with as little as 15% overlap
